# DEVELOPMENT RULES

## General

Never hardcode.

Always reuse.

Always document.

---

## Database

Never change migration without updating DATABASE.md.

Never delete column without approval.

Never rename table directly.

---

## UI

Dark Mode required.

Responsive required.

Use Flux UI.

No inline styling.

---

## Code

Use Service Layer.

Use Validation.

Use Enum when possible.

Avoid duplicated logic.

---

## Documentation

Every feature must update:

FEATURE.md

TODO.md

CHANGELOG.md

---

## Git

Small Commit.

One Feature One Commit.

Never commit .env.

Never commit database backup.

---

## AI

Read INDEX.md first.

Never scan whole project if FILEMAP.md is enough.
